*Bait post para o artigo "O software que vem não é menos software"*

---

Essa lógica chega até mim com frequência, nas equipes que lidero, nos clientes e nos parceiros.

Se a máquina produz código mais rápido, precisamos de menos pessoas para produzir código.

A lógica parece óbvia. E no curto prazo, parte dela se confirma: algumas funções estão comprimindo, o mercado está se ajustando.

Mas ela parte de uma premissa que não aguenta o exame no longo prazo: que existe uma quantidade fixa de software a ser construído.

Software nunca foi uma quantidade fixa. O que está acontecendo agora não é só compressão: é explosão. E uma mudança de natureza. Cada sistema novo que entra em produção muda o contexto em que os outros operam. Agentes autônomos, pipelines que se reconfiguram, código que escreve código. Cada composição gera estados que ninguém previu.

O mundo não está ficando mais simples com a IA. Está ficando mais complexo a cada sistema adicionado.

E num ambiente assim, a pergunta que importa não é "haverá menos desenvolvedores?": é "o que se torna escasso quando construir custa minutos?"
