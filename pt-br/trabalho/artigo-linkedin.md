## Fio condutor

O medo comum parte de uma premissa errada. A lógica é: AI escreve código mais rápido → menos trabalho de escrever código → menos desenvolvedores. Isso trata software como uma quantidade fixa de trabalho sendo distribuída entre máquinas e humanos.

Mas a premissa de quantidade fixa é a parte errada.

O que está acontecendo não é que o mesmo software está sendo feito mais rápido. É que o custo de construir software colapsou — e isso mudou o que está sendo construído. Saímos de software que executa estados para software que gera estados: sistemas agênticos, composições que produzem comportamento emergente, software que opera sobre outro software. O objeto do trabalho mudou de natureza.

Isso não é menos complexidade. É complexidade de outra ordem.

---

## Estrutura proposta

**1. Abertura: o medo errado**
O que todo mundo está sentindo — e por que a premissa está errada. Software não está desaparecendo. Está mudando de natureza.

**2. A virada: software que gera estados**
O software que vem não é menos software — é software de outra ordem. Cada sistema agêntico construído adiciona comportamento ao ambiente em que outros sistemas operam. Composições geram estados que ninguém previu. O território cresce enquanto é navegado.

**3. O que isso muda para quem constrói**
Navegar esse ambiente exige capacidade de resposta proporcional à sua variedade. AI reduz variedade — escolhe a interpretação mais provável e executa. É excelente nisso. Mas reduzir variedade num ambiente que cresce combinatorialmente não é navegar: é executar com confiança crescente num mapa que está ficando desatualizado.

**4. O modelo: desambiguação deliberada**
Introduzir o modelo mental brevemente — as duas dimensões, o scout, o diagnóstico. Não desenvolver tudo: abrir o apetite.

**5. Fechamento + referência ao GitHub**
O ativo mais valioso não é a velocidade de formalização — AI já resolveu isso. É a capacidade de identificar o que ainda não está resolvido, e saber o que fazer com isso.
