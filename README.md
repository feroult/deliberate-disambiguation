# From Deliberate Discovery to Deliberate Disambiguation

A mental model for software development as deliberate navigation under ambiguity.

## What it is

This repository contains the paper "Do Deliberate Discovery à Desambiguação Deliberada" — an extension of Dan North's Deliberate Discovery that names the central mechanism of software development: the continuous translation from natural language to formal language.

The paper proposes that software development is not construction — it is navigation. Teams don't know what they're building before they build it because there are two types of ambiguity that only become visible during the process: ignorance about the world (Dimension 1) and imprecision of language (Dimension 2). Deliberate Disambiguation is the stance of making this process explicit and intentional.
